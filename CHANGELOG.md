# Changelog

## 0.2.0
- **Pure-Python restructuring**: replaced the fragile Hugging Face Kaggle training path with a runnable calibration/tracing trainer
  - removed runtime dependence on `transformers`, `trl`, `peft`, and `bitsandbytes` for the training bundle
  - kept the kernel architecture intact while moving the runnable training path to deterministic JSON artifacts
  - added / retained a safe Kaggle flow that generates synthetic samples, trains lightweight calibrators, and saves inspectable outputs
- **Kaggle compatibility cleanup**:
  - documented the `trl` API drift (`max_seq_length` → `max_length`, `tokenizer` → `processing_class`, packing/text-field relocation)
  - documented the low-VRAM fallback from Qwen 7B to Qwen 1.5B
  - documented the fp16-only path after bf16 / CUDA extension failures
  - documented the `bitsandbytes` / CUDA runtime mismatch and why it was removed from the working path
- **Historical notes added**: the full debugging trail is now recorded in `docs/RESTRUCTURING_HISTORY.md`
- **Repo hygiene**: cleaned the push path so generated bytecode does not pollute the meaningful diff

## 0.1.1
- **Honesty fixes**: clarified that this is NOT a neural network implementation
  - added comprehensive docstrings explaining no weights, no PyTorch/JAX, no actual inference
  - renamed "preset" to "spec" throughout to clarify these are architecture specifications
  - updated README with explicit "What this IS vs. IS NOT" section
  - updated AGENTS.md with LLM module clarification
  - updated training/README.md to clarify simulation vs actual training
- Updated `KernelWeaveLLM` class docstring to clarify it's a routing layer, not a neural model
- Added `compact_frontier_spec()` and `reasoner_frontier_spec()` methods, deprecated old names
- Updated all documentation to reflect the actual state of the codebase
