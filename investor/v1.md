# V1 — Low-Sized Model Integration

## Goal
Attach a small model so KernelWeave can influence real inference, not just routed logic.

## Components
- low-sized model integration
- kernel-aware prompting
- promotion loop
- benchmark harness

## What this proves
- kernels improve actual model outputs
- kernel-aware prompts reduce mistakes
- promotion from trace to kernel is useful in production-like runs
- the architecture works on cheap hardware

## Prototype model direction
Use a small instruction model that fits on modest GPUs and is cheap to run.

## Benchmark harness
Measure:
- task accuracy
- verification pass rate
- latency
- token cost
- kernel reuse rate
