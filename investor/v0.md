# V0 — Minimum Working Prototype

## Goal
Prove the kernel architecture works on repeated tasks.

## Components
- trace capture
- kernel compiler
- kernel store
- router
- verifier

## Demo flow
1. Solve a task.
2. Record the successful trace.
3. Compile the trace into a kernel.
4. Store the kernel.
5. Route a similar prompt to the kernel.
6. Verify the output.

## V0 success criteria
- kernel reuse works
- outputs are verifiable
- routing is explainable
- store is persistent
- repeated prompts are cheaper than fresh generation
